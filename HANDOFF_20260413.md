## 미완료 작업

### 6. v1 (best_pst) eval — segfault 발생 ❌
Run: `20260413_040016_toronto_qap_refine_v1`
Checkpoint: `runs/stage2/20260413_040016_toronto_qap_refine_v1/checkpoints/best.pt`

**실패 상황**: Rochester 백엔드에서 alu-v0_27 (10Q, circuit 10/12) PST 시뮬레이션 중 segfault (exit 139).
- v2_normscore는 동일 백엔드에서 성공했으므로 v1 모델의 레이아웃이 더 깊은/복잡한 transpiled circuit을 생성하는 것으로 추정
- 분리 실행 필요: rochester만 먼저, 실패 시 washington만 별도 실행

**재시도 명령어:**
```bash
# Rochester만 먼저
conda run -n graphqmap python evaluate.py \
  --config configs/stage2_toronto_qap_refine.yaml \
  --checkpoint runs/stage2/20260413_040016_toronto_qap_refine_v1/checkpoints/best.pt \
  --backend rochester --reps 3 \
  --output runs/eval/20260413_040016_toronto_qap_refine_v1/best_pst/eval_results_rochester.csv

# Washington만
conda run -n graphqmap python evaluate.py \
  --config configs/stage2_toronto_qap_refine.yaml \
  --checkpoint runs/stage2/20260413_040016_toronto_qap_refine_v1/checkpoints/best.pt \
  --backend washington --reps 3 \
  --output runs/eval/20260413_040016_toronto_qap_refine_v1/best_pst/eval_results_washington.csv
```

**segfault 대안**: evaluate.py에 tensor_network 실패 시 simulator 재생성 로직이 있음.
Rochester segfault가 반복되면 alu-v0_27 회로를 건너뛰거나 simulation method를 변경 검토.

### 7. 결과 합치기
v1, v2 모두 rochester/washington 결과 확보 후, 기존 toronto 결과와 합쳐서
3-backend 종합 결과 CSV/summary를 생성해야 함.

예시:
```bash
# 3개 백엔드 결과를 합친 전체 eval
conda run -n graphqmap python evaluate.py \
  --config configs/stage2_toronto_qap_refine.yaml \
  --checkpoint runs/stage2/<RUN>/checkpoints/best.pt \
  --backend toronto rochester washington --reps 3
```
이렇게 하면 runs/eval/<RUN>/ 에 3-backend 통합 결과가 자동 생성됨 (기존 결과 덮어쓰기 주의).
