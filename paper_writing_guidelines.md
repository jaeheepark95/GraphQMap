# 논문 작성 가이드라인

본 문서는 논문 작성 시 준수해야 할 규칙과 스타일 지침을 정리한 것이다.

---

## 0. 최우선 스타일 규칙

아래 규칙은 다른 모든 문체 규칙보다 우선한다.

### 0.1 금지 단어

다음 단어는 논문 본문에서 사용하지 않는다.

- `with`
- `via`
- `through`
- `since`
- `across`
- `among`
- `leverage`
- `yield`
- `consequently`
- `under`
- `over`
- `so`
- `yet`

적용 범위는 Abstract, Introduction, Related Work, Method, Analysis, Discussion, Conclusion, 표 캡션, 그림 캡션, 표 내부 텍스트를 포함한다. 참고문헌 제목, 인용 원문, LaTeX 주석은 예외로 둔다.

### 0.2 비인칭 주어

논문 문장은 비인칭 주어를 기본으로 사용한다.

- `we`, `I`, `our`, `us`, `my` 같은 사람 지칭 주어는 사용하지 않는다.
- 권장 주어는 `this paper`, `the manuscript`, `the analysis`, `the theorem`, `the framework`, `the result` 등이다.
- 가능하면 문장 주어를 개념, 분석 대상, 정리, 방법론으로 둔다.

### 0.3 건조하고 중립적인 톤

문장은 건조하고 중립적인 톤으로 유지한다.

- 주장 과장은 피한다.
- 감정적 표현, 홍보성 표현, 수사적 강조는 피한다.
- novelty 강조보다 문제 정의, 가정, 정리, 한계, 범위를 우선한다.
- 가치판단형 표현보다 기술적 서술을 우선한다.

### 0.4 콜론·세미콜론 금지

논문 본문에서 콜론(`:`)과 세미콜론(`;`)은 사용하지 않는다.

- 콜론 대신: 문장을 분리하거나, 접속사("that is", "namely", "as follows")로 대체한다.
- 세미콜론 대신: 마침표로 문장을 분리하거나, 접속사("and", "but", "whereas")로 연결한다.
- **예외**: 수식 내부, 알고리즘 의사코드, 참고문헌 블록은 허용한다.

### 0.5 학술적인 톤

문장은 학술적인 톤으로 작성한다.

- 과도하게 장식적인 표현은 피한다.
- 지나치게 무겁거나 과장된 문체도 피한다.
- 정확성, 명료성, 절제된 전문성을 우선한다.

## 1. 금지 단어

다음 단어들은 논문 본문에서 사용하지 않는다 (참고문헌 제목 제외):

| 금지 단어                    | 권장 대체어                                                  |
| ---------------------------- | ------------------------------------------------------------ |
| **with**                     | using, at, having, and, offering, for, containing, involving, equipped by |
| **via**                      | by, using, by means of                                       |
| **through**                  | by, using, over, to, during                                  |
| **since**                    | because, as, given that                                      |
| **across**                   | over, spanning, in, between, throughout, by                  |
| **leverage** / **leverages** | exploit, use, employ, harness                                |
| **among**                    | of, in, between, one of                                      |
| **yield** / **yields**       | produce, provide, give, lead to, result in                   |
| **consequently**             | therefore, as a result, for that reason                      |
| **under**                    | in, within, subject to, for, at                              |
| **over**                     | across, throughout, during, for, in                          |
| **so**                       | therefore, thus, and, which means                            |
| **yet**                      | however, still, but, although                                |
| **so that**                  | and therefore, which gives, ensuring that, allowing          |
| **upon**                     | on, after, when, at                                          |

### 적용 범위

- 본문 전체 (Abstract, Introduction, Related Work, Method, Analysis, Experiments, Conclusion)
- 표 내용, 그림 캡션 포함
- **예외**: `\bibitem` 블록 내 참고문헌 제목, LaTeX 주석

### 치환 시 주의사항

- 문맥에 맞는 자연스러운 대체어 선택
- 기계적 치환 금지 — 문장 재구성이 필요할 수 있음
- 수학 표기 내 단어는 해당하지 않음 (예: 수식 설명의 "with d = 2^n"은 "where d = 2^n"으로 변경)

## 2. 볼드체 사용 규칙

### 허용

- 테이블 열 헤더 (`\textbf{Method}`, `\textbf{Qubits}` 등) — IEEE 표준 형식

### 금지

- 본문 내 용어 강조 목적의 볼드 (예: `\textbf{Staged Invariant Certification (SIC)}`)
- 비교 테이블에서 자기 방법(Ours) 행 전체를 볼드 처리하는 것
- 실험 설정에서 기준 방법 이름을 볼드 처리하는 것

### 허용되는 강조 방식

- `\emph{}` (이탤릭) — 최초 정의 시 사용 (예: `\emph{semantic preservation}`)
- 수학 기호 — 별도 강조 불필요

## 3. 초록 (Abstract) 작성 규칙

### 구조

초록은 **최대 8문장**으로 제한한다. 전체 구조는 다음 네 덩어리를 따른다.

1. **선행사 연구 / 배경**: 기존 맥락과 연구 대상의 위치를 제시한다.
2. **문제제기**: 현재 한계나 해결되지 않은 문제를 짧고 분명하게 제시한다.
3. **연구주제**: 논문이 다루는 핵심 주제와 제안의 방향을 설명한다.
4. **연구의의**: 이론적 의미, 해석 가능성, 범위, 필요시 실험적 결과의 의미를 간단히 제시한다.

이 구조는 세종대왕의 훈민정음 서문처럼, 배경을 먼저 두고 문제를 분명히 제시한 뒤, 해결 대상을 밝히고, 마지막에 의의를 정리하는 흐름을 따른다.

### 스타일

- 방법론이나 이론 세부를 초록에서 과도하게 설명하지 않는다.
- 독자가 본문 진입 전에 쉽게 이해할 수 있도록 평이하게 작성한다.
- 문장 수를 늘리기보다 구조적 선명성을 우선한다.
- 필요할 때만 실험적 결과의 의의를 마지막에 짧게 덧붙인다.
- 키워드는 최대 5개로 제한한다.

## 4. 서론 (Introduction) 작성 규칙

### 구조 (3~4문단 + 마지막 구성 문단)

1. **1문단**: 연구 배경 및 중요성
2. **2문단**: 선행 연구의 한계
3. **3문단**: 연구 목적, 동기, 필요성
4. **4문단(선택 또는 권장)**: 연구의 차별성, 범위, 기여의 성격
5. **마지막 짧은 문단**: 논문 전체 구조를 간략하게 설명한다.

### 문단 길이

- 각 문단은 **최소 7문장**, **최대 12문장**으로 작성한다.
- 3문단 구성이 가능하면 3문단으로 유지할 수 있다.
- 차별성과 범위가 중요하면 4문단 구성을 우선한다.

### 스타일

- `\IEEEPARstart` 사용하지 않음 — 일반 텍스트로 시작
- `this work` 대신 `this paper` 또는 비인칭 구조 사용
- 각 문단에는 적절한 참고문헌을 배치한다.
- 이탤릭체와 볼드체를 남발하지 않는다.
- 기여 목록과 논문 구성 요약은 필수 요소로 두지 않는다.
- 문제 정의, 선행 한계, 목적, 차별성이 문단 구조에서 자연스럽게 드러나야 한다.
- 마지막 구성 문단은 짧게 유지하며, 절 구성만 간략히 요약한다.

---

## 5. Related Work 작성 규칙

### 구조

- 1~2문단으로 작성한다.
- 각 문단은 최소 6문장으로 작성한다.
- 해당 연구주제에 맞추어 방법론, 이론, 검증 관점 등으로 그룹화하여 설명한다.
- 단순 나열보다 비교 가능한 묶음으로 정리한다.

### 내용 원칙

- 각 그룹은 대표 연구와 핵심 접근을 먼저 설명한다.
- 각 그룹은 한계점이나 적용 제약을 비판적으로 함께 적는다.
- 마지막에는 본 연구와의 차별성이 분명하게 드러나야 한다.
- 본 연구에 실험이 포함되어 있다면, 실험 비교대상 중심으로 우선 서술한다.

### 스타일

- 서술형 문단으로 작성한다.
- 단순 연대기식 정리는 피한다.
- 방법, 보장 범위, 적용 가능성, 비용 구조 같은 비교 축을 분명히 둔다.

---

## 6. Conclusion 작성 규칙

### 구조

- 한 문단으로 작성한다.
- 전체 길이는 6~10문장으로 제한한다.
- 본 논문의 핵심과 결과를 짧고 명확하게 요약한다.
- 시사점과 기여도를 분명하게 드러낸다.
- 필요할 때만 향후 연구를 마지막에 짧게 덧붙인다.

---

## 6. 일반 작성 규칙

### section / subsection / subsubsection 규칙

- section과 subsection은 여러 문단으로 작성할 수 있다.
- 다만 문단 수가 과도하게 분절되지 않도록 조절한다.
- subsubsection은 하나의 문단으로 작성한다.

### 기호와 notation 규칙

- 수학 또는 이론 중심 연구에서는 기호와 notation을 표 또는 본문에서 명확하게 정의한다.
- 새로운 기호는 사용 전에 먼저 소개한다.
- notation table이 있으면 핵심 기호를 우선 배치한다.

### 수식

- 중요 수식에는 `\label`과 번호 부여
- 인라인 수식은 `$...$` 사용
- 독립 수식은 `\begin{equation}` 또는 `\begin{align}` 사용
- 독립 수식이 길어 여러 줄이 되면서 하나의 번호를 공유해야 할 때에는 `\begin{split}`와 함께 사용한다.
- 프로베니우스 노름: `\|\cdot\|_F`

### 표

- 열 헤더만 볼드
- `\toprule`, `\midrule`, `\bottomrule` 사용 (booktabs 패키지)
- 자기 방법 행에 볼드 강조 금지
- PDF 기준으로 표가 column 폭을 넘으면 크기, 열 간격, 줄바꿈, 배치를 조정해 가독성을 확보한다.

### 참고문헌

- 참고문헌 제목 내 금지 단어는 원문 유지
